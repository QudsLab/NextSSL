# Binary Distribution

Pre-built binaries are distributed under `bin/` and indexed by
`bin/bins.json`.

---

## Directory Layout

```
bin/
  bins.json          ← master index of all available binaries
  windows/
    base/            ← standalone single-algorithm helpers
    main/            ← category-level binaries
    partial/
      core/          ← partial layer (core primitives)
      dhcm/          ← DHCM subsystem
      hash/          ← hash partials
      pow/           ← PoW partials
      pqc/           ← PQC partials
  linux/             ← (same structure)
  mac/               ← (same structure)
  web/               ← WebAssembly targets
```

---

## Extension Map

| Platform | Extension | Notes |
|---|---|---|
| Windows | `.dll` | Requires `-lbcrypt` at link time |
| Linux | `.so` | ELF shared object |
| macOS | `.dylib` | Mach-O dynamic library |
| WebAssembly | `.wasm` | Requires `wasmtime` or browser runtime |

---

## Tier Sizes (approximate)

| Tier | Description | Typical size |
|---|---|---|
| `partial` | Single primitive | 20–100 KB |
| `base` | Algorithm family | 80–300 KB |
| `main` | Category-level | 200–800 KB |
| `lite` | 9 core algorithms unified | ~500 KB |
| `full` | All algorithms unified | ~5 MB |

## Download URL Format

```
https://github.com/QudsLab/NextSSL/releases/download/<tag>/<stem>.<ext>
```

Example:

```
https://github.com/QudsLab/NextSSL/releases/download/v0.0.1-beta/nextssl_windows_full.dll
```

---

## `bins.json` Schema

The index is generated by `script/gen/gen_index.py` and contains 225 entries.

Keys are **stems without extension** — the platform and extension are
implicit from the directory context.

```json
{
  "nextssl_windows_full": {
    "platform": "windows",
    "tier": "main",
    "variant": "full",
    "url": "https://github.com/QudsLab/NextSSL/releases/download/v0.0.1-beta/nextssl_windows_full.dll",
    "sha256": "..."
  },
  ...
}
```

| Field | Type | Description |
|---|---|---|
| `platform` | string | `windows`, `linux`, `mac`, `web` |
| `tier` | string | `partial`, `base`, `main` |
| `variant` | string | `full`, `lite`, or algorithm name |
| `url` | string | Direct download URL |
| `sha256` | string | Hex SHA-256 of the binary |

## Checksums

Each release includes a `checksums.sha256` file alongside the binaries.

**Verify on Linux/macOS:**

```sh
sha256sum --check checksums.sha256
```

**Verify on Windows (PowerShell):**

```powershell
Get-FileHash nextssl_windows_full.dll -Algorithm SHA256
```

The `checksums.py` script in the repository root can also regenerate and
verify the checksum manifest:

```sh
python checksums.py verify bin/
```
