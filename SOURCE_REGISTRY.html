Table A
C Implementation Sources
Trustworthy, compilable C (and where noted C++/asm) repositories. Prioritised for: clean build systems, no exotic dependencies, wide platform support, and authoritative provenance. Each has a unique ID used in Table C.
Compilation Priority

For a new algorithm, try in order: C04 (libsodium) → C01 (OpenSSL) → C06/C09 (PQClean/liboqs for PQC) → C07 (XKCP for Keccak family) → C03 (Botan for broad coverage) → C05 (libtomcrypt for legacy). Assembly is acceptable from C07, C08, C13, C15, and C18.

Table B
Verification Sources
NIST ACVP vectors and Python libraries used to cross-verify C implementation output. Python is suitable here specifically for its verification role — not as a KAT target itself, but as the oracle you run your C output against.
Verification Workflow

Generate KAT output from your C implementation → pipe to Python verifier → cross-check against NIST ACVP (V01) or Wycheproof (V11) vectors. For PQC: always use V12 (NIST KAT) as ground truth, V06 (pyoqs) as the Python oracle. For ZK: V14 + V09 (SageMath).
